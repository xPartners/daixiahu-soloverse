# ybA2A

> 面向医保基金监管与经办场景的智能化规则治理引擎，基于生成式 AI 与医保垂直大模型。

## 项目简介

ybA2A 解决当前医保审核规则人工维护效率低、区域政策碎片化适配难、新型骗保手段响应滞后等行业核心痛点。

项目实现医保政策从自然语言文档到可执行审核规则的全自动化转化、反欺诈规则自主闭环演化、异地就医规则双向智能适配，可无缝嵌入现有医保智能审核体系，为医保经办机构、信息化厂商提供可落地的规则智能化升级方案。

## ❗ 核心行业痛点

- **规则人工运维成本高**：全国及各地医保政策、目录调整、审核条款海量，人工梳理、编码、更新周期长达数周，人力成本高且易出现规则疏漏。
- **区域政策碎片化严重**：各省 / 市医保待遇、支付范围、审核标准差异大，异地就医场景下规则双向适配难度高，结算争议频发。
- **反欺诈规则迭代滞后**：新型骗保手段不断涌现，人工发现线索 → 分析模式 → 制定规则 → 上线生效链路长，基金监管存在明显窗口期。
- **规则上线缺乏验证手段**：新规则发布前无法精准评估对基金收支、医疗机构的影响，易引发执行偏差与结算纠纷。

## ✨ 核心功能

### 1. 医保政策自动解析与规则结构化转化

- 支持 PDF / Word / 网页 / 公文扫描件等多格式医保政策文件、目录调整通知、新规条款的智能解析。
- 基于医保垂直大模型与领域 NLP 技术，自动抽取出待遇政策、报销比例、支付限制、目录范围、诊疗规范等核心规则要素。
- 自动转化为标准化可执行规则（支持 JSON / DSL 格式），可直接导入主流医保智能审核系统。
- 规则抽取准确率 ≥ 95%，规则上线周期从周级缩短至小时级。

### 2. 反欺诈规则自主演化与仿真验证

- 基于历史违规结算数据、实时骗保线索，AI 自动识别新型违规模式，自动生成候选反欺诈规则。
- 内置仿真推演引擎，基于历史全量结算数据模拟新规则运行效果，输出基金追回预估、误判率、影响机构数、风险等级等量化指标。
- 支持人工复核 + 一键上线，形成「线索发现 → 规则生成 → 仿真验证 → 上线生效 → 效果反馈」的闭环演化链路。
- 新型骗保行为的规则响应速度提升 80% 以上。

### 3. 异地就医规则双向智能适配

- 内置全国各省 / 市医保政策规则库，自动识别参保地与就医地的政策差异点。
- 异地就医结算时，自动双向适配两地政策规则，实时计算诊疗项目合规性、报销比例、支付限额与基金分担比例。
- 自动生成规则适配说明与差异对比，减少结算争议，提升异地就医经办效率。
- 覆盖普通门诊、住院、门诊慢特病、生育等全场景异地就医规则适配。

## 核心技术栈

- **大模型基座**：兼容 Llama 3、Qwen、GLM 等主流开源大模型，同时支持闭源 API 接入。
- **NLP 能力**：领域实体抽取、关系抽取、文档结构化解析、政策语义匹配。
- **规则引擎**：自研医保领域 DSL 规则语言，兼容主流商业规则引擎格式。
- **后端框架**：FastAPI + Python 3.10+。
- **数据存储**：PostgreSQL（结构化数据）+ Elasticsearch（全文检索）+ MinIO（文件存储）。
- **部署方案**：Docker + Kubernetes，全栈支持私有化部署，满足医保数据安全合规要求。

## 🚀 部署与启动

### 环境要求

- Python 3.10+
- （可选）Docker 与 Docker Compose
- （可选）PostgreSQL、Elasticsearch、MinIO（缺失时自动降级为无 DB 模式，不阻断启动）

### 配置文件

从模板复制并编辑配置：

```powershell
copy .env.example .env
notepad .env   # 或用 IDE 打开
```

**必填项（仅 1 个）**：

| 配置项 | 说明 |
|---|---|
| `LLM_API_KEY` | 大模型密钥（OpenAI / 通义千问 / 智谱 / 本地 Ollama 均可） |

**按需修改项**（本地开发默认即可）：

| 场景 | 修改内容 |
|---|---|
| 使用阿里通义千问 | `LLM_PROVIDER=qwen`、`LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1` |
| 使用智谱 GLM | `LLM_PROVIDER=glm`、`LLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4` |
| 使用本地 Ollama | `LLM_PROVIDER=ollama`、`LLM_BASE_URL=http://localhost:11434/v1` |
| Docker 部署 | 无需改动，`docker-compose.yml` 已自动将 `localhost` 映射为容器服务名 |

### 启动方式

**方式一：Docker Compose 一键启动（推荐）**

```powershell
docker-compose up -d --build
```

自动拉起应用 + PostgreSQL + Elasticsearch + MinIO 全套服务。

**方式二：本地开发启动**

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

> 应用启动时自动建表；数据库连接失败会降级为无 DB 模式，不影响 API 与前端使用。

| 入口 | 地址 | 适用对象 |
|---|---|---|
| **前端测试台** | `http://localhost:8000/app` | **医疗工作者**（可视化界面，中文卡片展示） |

### 前端测试台使用流程

1. 打开 `http://localhost:8000/app`（如页面异常，加版本号强制刷新：`http://localhost:8000/app?v=1`）
2. 左侧选择功能模块，右侧填写表单（已预填示例数据，可直接点击按钮测试）
3. 点击提交按钮后，下方显示实时进度（步骤、耗时），完成后展示中文卡片结果

### 五大功能模块

| 模块 | 操作 | 输出 |
|---|---|---|
| **政策解析** | 上传 PDF/Word/文本文件 + 填行政区划 | 自动识别格式，输出结构化审核规则（中文卡片） |
| **规则评估** | 提交结算单 JSON | 执行规则引擎，返回命中违规列表 |
| **反欺诈分析** | 输入历史结算数据 | AI 识别违规模式，生成候选反欺诈规则 |
| **仿真验证** | 输入待上线规则 + 历史数据 | 输出命中率、追回金额、误判率等量化指标 |
| **异地就医** | 指定参保地 + 就医地 | 双向适配两地政策，输出差异对比报告 |

### 支持的文件格式

| 格式 | 支持情况 | 说明 |
|---|---|---|
| PDF（文本型） | ✅ 完整支持 | 可选文字的 PDF |
| Word（.docx） | ✅ 完整支持 | Office 2007+ 格式 |
| 纯文本（.txt） | ✅ 完整支持 | 自动识别 UTF-8 / GBK / GB18030 编码 |
| Word（.doc 老格式） | ❌ 暂不支持 | 建议另存为 .docx |

> 文件格式**自动识别**，无需手动指定。解析失败时终端日志会输出具体归因（如"疑似扫描型 PDF，暂不支持 OCR"）。

## 🎯 应用场景

- **医保经办机构**：实现政策规则自动化运维，提升基金监管响应速度，降低异地就医结算争议。
- **定点医疗机构**：实时同步医保规则更新，诊疗前自动校验合规性，减少医保拒付风险。
- **商业健康保险**：快速适配各地医保政策，提升商保理赔审核效率与精准度。
- **医保信息化厂商**：嵌入现有产品体系，低成本快速提升产品智能化竞争力。
